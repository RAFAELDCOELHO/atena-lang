<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted Atena source → analyzer tree-walk | The analyzer recurses over a Program AST produced from arbitrary learner source text. Malformed or adversarially-nested programs could trigger unbounded recursion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | SemanticAnalyzer._visit (recursive tree-walk) | accept | Teaching programs are bounded in depth. Python's default recursion limit (~1000 frames) is sufficient for any realistic Atena program; a pathologically-nested expression (hundreds of levels) would hit Python's RecursionError, which the Phase-5 driver's top-level BaseException guard converts to a plain-English message (already designed in Phase 0). No additional guard needed in v1.0; documented limitation. |
| T-03-02 | Tampering | IndexAccess.index_converted flag | mitigate | The idempotency flag is set exactly once (guarded by `if node.index_converted: return` at the top of visit_IndexAccess). Tests assert the flag is True after analysis and that a second analysis pass does not shift the index again. |
| T-03-SC | Tampering | No package installs in this plan | accept | Plan 01 creates only Python source files — no npm/pip/cargo installs. No supply-chain surface. |
</threat_model>
