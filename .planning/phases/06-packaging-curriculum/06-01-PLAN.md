## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pyproject.toml metadata | Packaging metadata is read by pip/hatchling at build time; incorrect metadata could mislead package consumers but does not affect runtime security |
| pip install (supply chain) | `pip install .` fetches hatchling as a build-time dep; hatchling is the only external tool invoked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | pyproject.toml `dependencies = []` | mitigate | The plan explicitly requires `dependencies = []` to be preserved unchanged; Task 2 `pip show atena-lang` asserts "Requires:" is blank — any accidental addition is caught at verification |
| T-06-02 | Tampering | hatchling (build-time dep) | accept | hatchling is already declared in Phase 0 and is a well-known PyPA tool with no runtime footprint; supply chain risk is the same as Day 0 |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | Only `pip install -e .` is run; no new packages are added to `dependencies`; hatchling is already the declared build backend. No new packages introduced in this plan. |

No HIGH-severity threats are present. This plan has no network I/O, no auth, no persisted user data, no web surface. The only security consideration is the supply-chain invariant (zero runtime deps) which is enforced by an explicit verification assertion.
</threat_model>
