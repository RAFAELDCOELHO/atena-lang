<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → argparse | User-supplied filename and flags; argparse validates structure |
| Generated Python string → exec() | Output of the transpiler pipeline — built from a typed AST, never string-interpolated from user source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-A | Elevation of Privilege | exec() of generated Python | accept | Local-only: the learner runs their own code on their own machine. Generated Python comes from a typed AST via ast.unparse() — no f"…{user_source}…" interpolation path exists. PITFALLS.md confirms no injection vector. |
| T-05-02-B | Information Disclosure | Exception from exec() leaking to stderr | mitigate | BaseException catch in cli.py wraps all runtime errors; no raw traceback or class name reaches stderr. Plan 03 adds curated messages; this plan keeps the existing _internal_error_message as a temporary guard. |
| T-05-02-C | Tampering | --show flag writing to stdout | accept | Print-only; no filesystem write from --show. The build path already requires write access to the output directory, which is a separate concern. |
| T-05-02-D | Denial of Service | Learner program with infinite loop in exec() | accept | No timeout implemented in v1.0 (out of scope). The learner's own program — consistent with running any user-authored Python script. |
| T-05-02-SC | Tampering | No new package installs | accept | stdlib only; no pip install steps; slopcheck N/A |
</threat_model>
