Symbol table model (from D-07, 03-PATTERNS.md §constructor):
  self._globals: dict[str, str]       # name → inferred type (top-level scope)
  self._locals: dict[str, str] | None # set (non-None) while inside a FunctionDef body
  self._functions: dict[str, int]     # function-name → arity (param count)
  self._current_fn: str | None        # name of enclosing function, or None

Scope resolution rule (D-07):
  scope = self._locals if self._locals is not None else self._globals
  Lookup order when inside a function (self._locals is not None):
    1. self._locals  (params + local assignments)
    2. self._functions (can call earlier-defined functions from inside a function)
    3. NOT self._globals (top-level variables are forbidden — D-07 pure function)
  Lookup order at top level (self._locals is None):
    1. self._globals (which also contains function names registered as "function" type)

visit_Identifier (undefined detection + poisoning + D-08 outer-var):
  name = node.name
  # 1. Check current scope first
  scope = self._locals if self._locals is not None else self._globals
  if name in scope:
      return scope[name]
  # 2. If inside a function, check self._functions (can call earlier-defined functions)
  if self._locals is not None and name in self._functions:
      return "unknown"   # it's a callable, not a value — type is irrelevant here
  # 3. If inside a function, check if name exists at top level (D-08 tailored message)
  if self._locals is not None and name in self._globals:
      self._errors.add(node.line,
          f'A function can only use its own inputs — pass "{name}" in as a parameter.',
          node.source_line)
      scope[name] = "unknown"   # poison in local scope to suppress cascade
      return "unknown"
  # 4. Fully undefined
  candidates = list(scope.keys()) + list(ATENA_KEYWORDS)
  hint = suggest(name, candidates)
  msg = f'I don\'t know what "{name}" is yet. Did you forget to create it first?'
  if hint:
      msg = f'{msg} {hint}'
  self._errors.add(node.line, msg, node.source_line)
  scope[name] = "unknown"   # poison: suppress cascade (D-09, PITFALLS 12)
  return "unknown"

visit_FunctionDef (scope push/pop, per 03-PATTERNS.md §FunctionDef):
  # Register BEFORE visiting body (self-call is allowed; no hoisting for external calls)
  self._functions[node.name] = len(node.params)
  self._globals[node.name] = "function"    # so top-level code can call it after this point
  # Push local scope
  saved_locals = self._locals
  saved_fn = self._current_fn
  self._locals = {p: "unknown" for p in node.params}   # params typed unknown (D-01 under-coerce)
  self._current_fn = node.name
  try:
      for stmt in node.body:
          self._visit(stmt)
  finally:
      self._locals = saved_locals
      self._current_fn = saved_fn
  return "unknown"

visit_FunctionCall (defined-before-called + arity, per 03-PATTERNS.md §FunctionCall):
  # 1. Visit args first (bottom-up)
  for arg in node.args:
      self._visit(arg)
  # 2. Pass-through builtins: never in _functions, never error
  BUILTINS = {"length", "str", "_atena_concat", "_atena_index"}
  if node.name in BUILTINS:
      return "unknown"
  # 3. Check defined-before-called
  if node.name not in self._functions:
      if node.name in self._globals:
          # It's a variable being called like a function
          self._errors.add(node.line,
              f'"{node.name}" is not a function — it\'s a value you stored. You cannot call it.',
              node.source_line)
      else:
          candidates = list(self._globals.keys()) + list(ATENA_KEYWORDS)
          hint = suggest(node.name, candidates)
          msg = f'I don\'t know a function called "{node.name}" yet — define it above this line first.'
          if hint:
              msg = f'{msg} {hint}'
          self._errors.add(node.line, msg, node.source_line)
      return "unknown"
  # 4. Arity check
  expected = self._functions[node.name]
  given = len(node.args)
  if expected != given:
      plural = "s" if expected != 1 else ""
      self._errors.add(node.line,
          f'"{node.name}" expects {expected} value{plural}, but you gave {given}.',
          node.source_line)
  return "unknown"

Single namespace for functions + variables (Claude's Discretion from CONTEXT.md):
  Both self._globals and self._locals use one dict, so a function name and a variable
  name cannot coexist with the same key. self._globals[name] = "function" registers
  the function; if a variable is later assigned the same name, the last-assignment-wins
  rule applies (it overwrites "function" with the new inferred type). The visit_FunctionCall
  check handles this: if name is in self._globals but NOT in self._functions, the "not a
  function" message fires. This is the single-namespace behavior matching generated Python.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted Atena source → scope resolution | A learner program may reference names in any order; the analyzer must handle undefined names without crashing or leaking a Python exception. |
| visit_FunctionDef body traversal | Function bodies may contain nested calls, undefined names, or mismatched arities — all must be collected without crashing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Denial of Service | visit_FunctionDef on deeply nested function calls within body | accept | Same recursion bound as T-03-01. Phase-5 driver handles RecursionError. |
| T-03-06 | Spoofing | _atena_concat / _atena_index names reserved as builtins in visit_FunctionCall | mitigate | BUILTINS set {"length","str","_atena_concat","_atena_index"} is checked first; a learner who writes a function named _atena_concat would be creating a name-collision that the analyzer correctly passes through without arity error, since learners cannot produce underscore-prefixed names through normal Atena syntax (the lexer produces them only via FunctionCall injection). Document as a reserved-name convention. |
| T-03-07 | Elevation of Privilege | try/finally scope restoration in visit_FunctionDef | mitigate | The finally block is unconditional — even if an internal Python exception escapes from body traversal (which should not happen in normal use since the analyzer catches via errors.add), the scope is always restored. No scope leak between functions or back to global. |
| T-03-SC | Tampering | No package installs in this plan | accept | Plan modifies only Python source files. No supply-chain surface. |
