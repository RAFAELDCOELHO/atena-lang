## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| malformed source text → _synchronize | worst-case adversarial: all-operator lines, missing keywords, deeply nested broken structures |
| identifier-shaped Python keywords → dispatcher | "def", "elif", "for", "class", "import" lex as IDENTIFIER and are indistinguishable from user names until checked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Availability | _synchronize() not making progress | mitigate | Backstop in _parse_statement(): if self._pos == pos_before after exception handling, force _advance() once — this covers the edge case where _synchronize() exits immediately without consuming (should not happen but defended) |
| T-02-14 | Availability | Infinite-error flood from one bad expression | mitigate | _parse_statement() catches _ParseError at statement boundary; _synchronize() skips to next NEWLINE/DEDENT; one bad statement = one error add() call; the dedup in ErrorCollector caps same-line same-message repeats |
| T-02-15 | Information Disclosure | "def"/"class" identifiers reaching _parse_expression | mitigate | Redirects added BEFORE the expression/assignment branch in _dispatch_statement; they fire first and never reach _parse_primary() |
| T-02-16 | Integrity | Top-level return silently producing a Return node | mitigate | _parse_return() checks fn_depth==0 and raises _ParseError BEFORE constructing the Return node; no partial AST node for an invalid return is inserted |
</threat_model>
