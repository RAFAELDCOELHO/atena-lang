  <action>
    Add the following methods to the Parser class in src/atena/parser.py. Keep the skeleton from Plan 01
    intact; only ADD new methods.

    Method order to implement:

    1. _parse_expression(self, min_bp: int = 0) -> Node:
       Pratt loop. Call _parse_unary() to get left. Then loop:
       - peek current token; extract op_str from its value (works for OPERATOR, COMPARISON, KEYWORD "and"/"or" tokens).
       - bp = _BINARY_BP.get(op_str, 0). If bp <= min_bp: break.
       - _advance() to consume the operator token (save as op_tok for line/source_line).
       - right = _parse_expression(bp)  ← left-associative: pass bp (not bp-1) so right sees same min_bp, meaning equal-precedence ops on right are rejected, giving left-assoc.
       - left = BinOp(op=op_str, left=left, right=right, line=op_tok.line, source_line=op_tok.source_line).
       Return left.
       Note on left-assoc: for left-associative, right operand must use bp (not bp-1). With Pratt,
       passing `min_bp=bp` to the right parse means operators of the SAME precedence on the right
       will NOT be absorbed (since they need bp > min_bp to bind), giving left-association. This is the
       correct implementation per PITFALLS.md §8. Verify: "10 - 3 - 2" → the second "-" has bp=4,
       and the right call runs with min_bp=4, so it cannot absorb another "-" (which also has bp=4,
       and 4 > 4 is false) → correct left-assoc.

    2. _parse_unary(self) -> Node:
       Check current token:
       - If it is KEYWORD with value "not": advance, call _parse_unary() recursively (right-recursive),
         return UnaryOp(op="not", operand=..., line=tok.line, source_line=tok.source_line).
       - If it is OPERATOR with value "-": advance, call _parse_postfix(_parse_primary()) for the
         operand (unary minus binds tighter than any binary — only applies to the immediate primary+postfix),
         return UnaryOp(op="-", operand=..., line=tok.line, source_line=tok.source_line).
       - Otherwise: return _parse_postfix(_parse_primary()).

    3. _parse_postfix(self, node: Node) -> Node:
       Tight loop binding [] . () left-associatively, tighter than any binary op.
       While True, check current token:
       - LBRACKET: advance (save as tok); parse index = _parse_expression(); _expect(RBRACKET,
         'I reached the end of the line still waiting for a "]".'); wrap as IndexAccess(target=node,
         index=index, index_converted=False, line=tok.line, source_line=tok.source_line); continue.
       - DOT: advance (save as tok); name_tok = _expect(IDENTIFIER, 'Expected a field name after ".".'); 
         wrap as DotAccess(target=node, name=name_tok.value, line=tok.line, source_line=tok.source_line); continue.
       - LPAREN: advance (save as tok); args = []; if not _check(RPAREN): args.append(_parse_expression());
         while _match(COMMA): args.append(_parse_expression()); _expect(RPAREN,
         'I reached the end of the line still waiting for a ")".'); if isinstance(node, Identifier):
         wrap as FunctionCall(name=node.name, args=args, line=tok.line, source_line=tok.source_line);
         else: raise _ParseError(tok.line, 'Only named functions can be called.', tok.source_line);
         continue.
       - Otherwise: break.
       Return node.

    4. _parse_primary(self) -> Node:
       Dispatch on current token type:
       - NUMBER: advance; return NumberLiteral(value=int(tok.value), line=tok.line, source_line=tok.source_line).
       - STRING: advance; return StringLiteral(value=tok.value, line=tok.line, source_line=tok.source_line).
         Note: tok.value for a STRING token is the literal content WITHOUT outer quotes (the lexer strips them).
         If the lexer does NOT strip quotes, strip here: tok.value.strip('"').
       - KEYWORD with value "true": advance; return BoolLiteral(value=True, line=tok.line, source_line=tok.source_line).
       - KEYWORD with value "false": advance; return BoolLiteral(value=False, line=tok.line, source_line=tok.source_line).
       - KEYWORD with value "length": this is a prefix unary-style keyword consuming the next expression.
         Save tok; advance; operand = _parse_primary() (length binds tightest — only to immediate primary,
         no postfix chain beyond it). Return FunctionCall(name="length", args=[operand], line=tok.line,
         source_line=tok.source_line). This maps to Python len() in codegen.
       - IDENTIFIER: advance; return Identifier(name=tok.value, line=tok.line, source_line=tok.source_line).
       - LPAREN: advance; expr = _parse_expression(); _expect(RPAREN, 'I reached the end of the line still waiting for a ")".'); return expr.
       - LBRACKET: return _parse_list_literal().
       - LBRACE: return _parse_dict_literal().
       - Anything else: raise _ParseError(tok.line, f'I didn\'t expect "{tok.value}" here.', tok.source_line).

    5. _parse_list_literal(self) -> ListLiteral:
       tok = _expect(LBRACKET, 'Expected "[" to start a list.').
       elements = []. If not _check(RBRACKET): elements.append(_parse_expression()); while _match(COMMA): elements.append(_parse_expression()).
       _expect(RBRACKET, 'I reached the end of the line still waiting for a "]".').
       Return ListLiteral(elements=elements, line=tok.line, source_line=tok.source_line).

    6. _parse_dict_literal(self) -> DictLiteral:
       tok = _expect(LBRACE, 'Expected "{" to start a dictionary.').
       pairs: list[tuple[str, Node]] = []. If not _check(RBRACE):
         key_tok = _expect(IDENTIFIER, 'Expected a key name in the dictionary.').
         _expect(ASSIGN, 'Expected "=" after the key name in the dictionary.').
         val = _parse_expression().
         pairs.append((key_tok.value, val)).
         While _match(COMMA):
           key_tok = _expect(IDENTIFIER, 'Expected a key name in the dictionary.').
           _expect(ASSIGN, 'Expected "=" after the key name.').
           val = _parse_expression().
           pairs.append((key_tok.value, val)).
       _expect(RBRACE, 'I reached the end of the line still waiting for a "}".').
       Return DictLiteral(pairs=pairs, line=tok.line, source_line=tok.source_line).
