# Phase 4 Plan 2: GREEN Phase — Core Construct Emitters + GEN-03/GEN-05 Gates

CodeGenerator.generate() implemented for all core constructs via ast.unparse() spine with D-02 post-patches and GEN-05 self-check.

## What Was Built

### Task 1: generate() and Core Construct Emitters (`src/atena/codegen.py`)

Implemented the full `generate()` entry point and all core emitters in `src/atena/codegen.py`:

**`generate()` body (D-01 spine):**
- Walks `self._program.statements`, calls `self._emit(stmt)` for each
- Calls `self._build_preamble()` (returns `[]` stub -- helper bodies in Plan 03) and prepends to body
- Builds `ast.Module(body=preamble + body_stmts, type_ignores=[])`
- Calls `ast.fix_missing_locations(module)` then `ast.unparse(module)`
- Applies three D-02 patches in order: `_patch_double_quotes`, `_patch_blank_lines`, `_patch_header`
- Runs GEN-05 self-check: `ast.parse(python_source)` -- SyntaxError propagates as internal bug
- Returns `python_source`

**D-02 patches:**
- `_patch_double_quotes`: `re.sub(r"'([^'\\]*)'", r'"\1"', src)` -- targeted regex, safe for strings with single-quote content (ast.unparse already uses double-quotes for those)
- `_patch_blank_lines`: `re.sub(r'\n(def )', r'\n\n\1', src)` -- readability patch for top-level functions
- `_patch_header`: prepends `# Generated by Atena\n`

**Core emitters implemented:**
- `_emit_Assign`: handles both plain assignment and dot-write (via `hasattr(node, "_dot_target")`)
- `_emit_Show` -> `print(value)`
- `_emit_Ask` -> `target = input("prompt")`
- `_emit_If`, `_emit_While` -> ast.If / ast.While
- `_emit_Repeat` -> `for _atena_i{counter} in range(count):` with monotonic counter
- `_emit_FunctionDef`, `_emit_Return`
- `_emit_ListAdd` -> `.append()`, `_emit_ListRemove` -> `.remove()`
- `_emit_FunctionCall` with `_used_helpers` tracking for `_atena_concat` / `_atena_index`
- `_emit_BinOp` dispatching 12 operators: `+,-,*,/` -> ast.BinOp; `==,!=,<,<=,>,>=` -> ast.Compare; `and,or` -> ast.BoolOp
- `_emit_UnaryOp`: `not` -> ast.Not(), `-` -> ast.USub()
- `_emit_ListLiteral`, `_emit_DictLiteral`, `_emit_IndexAccess`, `_emit_DotAccess`
- `_emit_Identifier`, `_emit_NumberLiteral`, `_emit_StringLiteral`, `_emit_BoolLiteral`
- `_build_preamble()`: on-demand helper emission (stub returning `[]` for now)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Arithmetic compound expressions routed through _atena_concat**
- Found during: Task 1 (test_G2_arithmetic_executes failing with `212` instead of `14`)
- Issue: `analyzer.visit_BinOp` returned `"unknown"` for all non-`+` operators (`-`, `*`, `/`). This caused `2 + (3 * 4)` to go through `_atena_concat(2, 12)` which does `str(2) + str(12)` = `"212"` instead of `14`.
- Fix: Added `if node.op in ("-", "*", "/"):` branch returning `"number"` before the non-`+` fallthrough. Arithmetic ops in Atena v1.0 (integers only) always produce numbers; returning `"number"` lets `number + number` take the no-coerce path.
- Files modified: `src/atena/analyzer.py`
- All 38 analyzer tests still pass after the fix; no regressions.
- Commit: `430f6b3`

**2. [Rule 1 - Bug] test_G1_if_else used undefined variable `x`**
- Found during: Task 1 (test_G1_if_else_generates_correctly failing with semantic error)
- Issue: Test snippet `"if x > 0\n    show x\nelse\n    show 0\n"` used `x` without declaring it, causing `SemanticAnalyzer` to emit an "I don't know what 'x' is yet" error. The `_generate()` helper asserts `ec.is_empty()` so the test failed before reaching codegen.
- Fix: Changed snippet to `"x = 5\nif x > 0\n    show x\nelse\n    show 0\n"` -- declares `x` first.
- Files modified: `tests/test_codegen.py`
- Commit: `430f6b3`
